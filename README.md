## Веб-сервис по агрегации блогов (PHP + MySQL + Docker)

**Функционал:**
- **Регистрация и вход** пользователей, таблица пользователей содержит поле `is_moderator`.
- **Создание, редактирование и удаление блогов** с привязкой к предопределённым темам (категориям).
- **Поиск** по заголовку и содержимому блогов и **фильтрация по категориям**.
- **Права доступа:** модераторы могут управлять любыми блогами, обычные пользователи — только своими.

### Технологический стек
- **Клиент:** HTML5 + CSS3 (простая адаптивная верстка).
- **Сервер:** PHP 8.2 (Apache) по паттерну **MVC**.
- **БД:** MySQL 8 (SQL-совместимая).
- **Инфраструктура:** Docker + `docker-compose`, сервисы: `nginx`, `php` (apache), `db` (mysql).

### Структура проекта
- `public/` — фронт-контроллер `index.php`, статика (`css`).
- `app/core/` — ядро: `Bootstrap` (роутинг), `Controller`, `Database`, `Auth`.
- `app/models/` — модели `User`, `Blog`, `Category` (CRUD, поиск, фильтрация, роли).
- `app/controllers/` — контроллеры `AuthController`, `BlogController`.
- `app/views/` — представления (layout, страницы авторизации, списка блогов, просмотра/редактирования/удаления).
- `db/init.sql` — SQL-скрипт создания схемы и начальных категорий.
- `docker-compose.yml`, `Dockerfile.php`, `nginx/default.conf` — инфраструктура.

### Запуск через Docker
1. Установите Docker и Docker Compose.
2. В корне проекта выполните:
   ```bash
   docker-compose up --build
   ```
3. Приложение будет доступно по адресу `http://localhost:8080`.

### Работа с приложением
- **Регистрация:** перейдите на `/?controller=auth&action=register`, создайте пользователя.
- **Вход:** `/?controller=auth&action=login`.
- **Список блогов:** главная страница `/?controller=blog&action=index` (по умолчанию).
  - В верхней части есть **поле поиска** и **список категорий** для фильтрации.
- **Создание блога:** авторизуйтесь и нажмите «Новый блог».
- **Редактирование и удаление:**
  - **Обычный пользователь** может редактировать/удалять **только свои** блоги.
  - **Модератор** (поле `is_moderator = 1` в таблице `users`) может управлять **любыми** блогами.

Чтобы сделать пользователя модератором, можно вручную обновить поле `is_moderator` для нужной записи в таблице `users` через любой SQL-клиент, подключённый к базе `blog_aggregator`.
